Kilo UDP Server
MMORPG foundation · Stage 1

Минимальный UDP сервер для прототипа

Сервер принимает и отдает UDP пакеты с данными игрока: id, name, x, y. Это стартовая точка для сетевого слоя MMORPG на Godot (C#).

Что делает

Слушает порт, парсит пакеты игроков, хранит игроков и карту в памяти и рассылает обновления.

Что не делает

Нет БД, комнат, NPC или валидации прав на изменение карты.

Формат пакета

id|name|x|y (UTF-8, разделитель — |).

Быстрый старт

Сервер готов к запуску сразу после сборки.

dotnet build

dotnet run --project KiloServer.csproj
# или с другим портом

dotnet run --project KiloServer.csproj -- 7777

Протокол

Запрос / Ответ

Клиент отправляет одну строку, сервер отвечает списком игроков.

player-1|Ari|12.5|8.75

Ответ содержит несколько строк (по одной на игрока).

player-1|Ari|12.5|8.75
player-2|Nova|4|9

Числа используют точку как разделитель.

Регистрация и вход

Регистрация создает аккаунт на сервере.

REGISTER|name|password

Вход проверяет пароль и резервацию имени.

LOGIN|name|password

Ответ на успешный вход содержит сохраненные координаты и уровень доступа.

OK|x|y|accessLevel

Для проверки доступности сервера используется пинг.

PING

Карта

Клиент запрашивает чанки карты по координатам чанка.

CHUNK|cx|cy

Ответ — JSON сетка тайлов 64x64.

{"cx":0,"cy":0,"size":64,"tiles":[...]}

Обновления карты отправляются с измененными тайлами.

MAPUPDATE|{"changes":[{"x":10,"y":5,"tile":1}]}

Сервер применяет изменения, сохраняет карту и рассылает MAPUPDATE всем клиентам.

Сохранение

Состояние игроков хранится в памяти и сохраняется в файл Data/players.json.

Аккаунты сохраняются в файл Data/accounts.json при регистрации.

Пароли не хранятся в открытом виде, используется соль и хэш.

Карта хранится в файле Data/world.json и обновляется при MAPUPDATE.

Файлы данных располагаются рядом с исполняемым файлом сервера.

Сохранение выполняется атомарно через временный файл.

Сохранение выполняется раз в 5 минут и при завершении сервера.

Игрок считается активным, если присылал пакет в последние 10 секунд.

Авторизация

Перед входом клиент отправляет LOGIN|name|password.

Ошибки: ERR|not_found, ERR|wrong_password, ERR|name_taken.

Успешный LOGIN резервирует имя на 10 секунд активности.

Пример отправки через PowerShell

$udp = New-Object System.Net.Sockets.UdpClient
$endpoint = New-Object System.Net.IPEndPoint([System.Net.IPAddress]::Parse("127.0.0.1"), 7777)
$payload = "player-1|Ari|12.5|8.75"
$bytes = [System.Text.Encoding]::UTF8.GetBytes($payload)
$udp.Send($bytes, $bytes.Length, $endpoint)

$remote = New-Object System.Net.IPEndPoint([System.Net.IPAddress]::Any, 0)
$reply = $udp.Receive([ref]$remote)
[System.Text.Encoding]::UTF8.GetString($reply)

Интеграция с Godot (C#)

Скетч для клиента. Это не полный класс, а базовая идея отправки.

var udp = new System.Net.Sockets.UdpClient();
var payload = "player-1|Ari|12.5|8.75";
var bytes = System.Text.Encoding.UTF8.GetBytes(payload);
await udp.SendAsync(bytes, bytes.Length, "127.0.0.1", 7777);

var result = await udp.ReceiveAsync();
var response = System.Text.Encoding.UTF8.GetString(result.Buffer);
