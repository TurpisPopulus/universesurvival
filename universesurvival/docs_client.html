Клиент: вход, меню, мир и редакторы

Клиент отвечает за UI, визуализацию мира и синхронизацию с UDP-сервером. Вся игра живет в сцене main.tscn, а взаимодействие с сервером происходит через короткие текстовые команды и JSON-снимки по версиям.

Godot 4.5

UI + Scene Главное меню и игровая сцена находятся в отдельных tscn.

UDP протокол

Регистрация, вход и синхронизация карты/объектов выполняются по UDP.

Редакторы

Внутри игры доступны админские панели редактирования карты и объектов.

Быстрый старт

  • Запусти сервер: server/run_server.bat.
  • Открой проект Godot и запусти сцену res://main_menu.tscn.
  • Используй Create Player для регистрации, затем Enter для входа.

Главное меню (main_menu.tscn + MainMenu.gd)

  • Enter открывает окно входа и отправляет LOGIN.
  • Create Player показывает форму регистрации и отправляет REGISTER.
  • Settings открывает экран настроек клиента.
  • Exit закрывает игру.

Окна входа/регистрации и настройки скрывают главное меню, чтобы не было наложений.

Настройки

Экран настроек содержит четыре кнопки: Видео, Звук, Авторы, Назад.

  • Видео: выбор доступного разрешения окна и переключение режимов Окно/Полный экран.
  • Звук: пока пусто (заготовка под будущие слайдеры громкости).
  • Авторы: краткая информация о проекте.
  • Назад: возврат в главное меню.

Список разрешений формируется из распространенных размеров, не превышающих размер экрана.

Игровая сцена (main.tscn)

  • WorldMap и WorldObjects отображают карту и объекты, WorldResources рендерит ресурсы в два слоя (низ/верх).
  • Player отвечает за движение и отправку позиционных пакетов.
  • Ui содержит админское меню и панели редакторов.
  • LoadingOverlay показывает загрузочный экран до полной синхронизации.
  • PauseOverlay открывается по ESC, ставит игру на паузу и содержит настройки.

Сеть и протокол

Клиент общается с сервером через UDP-сообщения:

Все сообщения идут в защищенном конверте SEC1:

SEC1|base64(version|nonce|ciphertext|mac)

Внутри шифрованного payload передаются команды:

PING
PONG
REGISTER|name|password|appearance
LOGIN|name|password
id|name|x|y|appearance
CHUNK|cx|cy|lastVersion
EDIT|{...}
OBJECTS|cx|cy|lastVersion
OBJECT_EDIT|{...}
RESOURCES|cx|cy|lastVersion
RESOURCE_EDIT|{...}
BLOCKING|cx|cy|lastVersion
BLOCKING_EDIT|{...}
SURFACE|cx|cy|lastVersion
SURFACE_EDIT|{...}
  • appearance - base64 JSON с внешностью, передается при регистрации/входе и в позиционных пакетах.
  • CHUNK и OBJECTS возвращают JSON-снимки только при изменении версии.
  • PING/PONG - проверка доступности сервера.
  • Клиент отправляет запросы чанков только пока версия не подтверждена (initial load, переход в новую область, локальные изменения).
  • Сервер применяет rate limiting на UDP-пакеты (по умолчанию 120 пакетов/сек, burst 200): лишние пакеты отбрасываются.

Безопасность и шифрование UDP

  • Все UDP-пакеты шифруются и подписываются: потоковый XOR на базе HMAC-SHA256 + HMAC-SHA256.
  • Пароли, appearance и координаты не уходят в открытом виде.
  • Ключ общий для клиента и сервера: universesurvival/NetworkCrypto.gd и server/Program.cs должны совпадать.
  • Ключ хранится в клиентском скрипте, поэтому его можно извлечь при реверсе.
  • Если MAC не совпадает, пакет считается поддельным и игнорируется.
  • Общий ключ можно извлечь из клиента; для полноценной защиты нужен обмен ключами и ротация.

Для публичных серверов используйте DTLS/Noise или отдельный безопасный канал для обмена ключами.

Внешность персонажа

Модуль CharacterAppearance.gd собирает paperdoll по наборам атласов:

  • Тело: body_skinny.png, body_normal.png, body_fat.png.
  • Головы, волосы, бороды, лицо: отдельные атласы с 6 кадрами анимации.
  • Настройки внешности сохраняются как base64 JSON в аккаунте.
  • Атласы предзагружаются через preload(), чтобы не блокировать старт в _ready().

Система тайлсетов (TilesetManager)

Клиент поддерживает работу с несколькими тайлсетами одновременно через глобальную систему ID:

  • TilesetManager.gd управляет преобразованием локальных ID тайлов в глобальные.
  • Глобальный ID = tileset_index * 100000 + local_tile_id (например: ground:52 = 100052).
  • Каждый тайлсет регистрируется с уникальным индексом при первом использовании.
  • Автоматическое сканирование: TilesetManager и TilePalette автоматически загружают все PNG файлы из папки tiles/.
  • При переключении тайлсетов в редакторе существующие тайлы на карте сохраняют свой исходный вид.

Как добавить новый тайлсет:

  • Создай PNG с тайлами размером кратным 32x32.
  • Положи файл в tiles/ (например, tiles/buildings.png).
  • Тайлсет автоматически появится в выпадающем списке редактора (имя = название файла без расширения).
  • TilesetManager автоматически присвоит ему следующий свободный индекс (сортировка по алфавиту).

Примечание: Файл data/tilesets.json больше не используется - все тайлсеты загружаются автоматически из директории.

Палитра тайлов (TilePalette)

TilePalette отображает текущий тайлсет и позволяет выбирать тайлы кликом:

  • Выпадающий список для переключения между доступными тайлсетами.
  • Текстура тайлсета отображается в ScrollContainer с вертикальной прокруткой.
  • Сетка тайлов вычисляется автоматически из размера текстуры (32x32 на тайл).
  • Клик по тайлу выбирает его для рисования и отправляет сигнал tile_selected.
  • Переключение тайлсета отправляет сигнал tileset_changed в WorldMap.
  • Выбранный тайл подсвечивается белой рамкой.

Если окно редактора маленькое, а тайлсет большой, автоматически появляется вертикальная прокрутка.

Редакторы карты и ресурсов

  • Админская кнопка открывает меню с двумя редакторами: Редактор карты и Ресурсы.
  • Редактор карты использует TilePalette.gd с поддержкой нескольких тайлсетов и панель статусов справа.
  • Панель статусов содержит кнопки для установки блокировок и типов поверхности: B (блокировка), S (мелководье), D (глубина), I (лёд), M (болото), очистка.
  • При открытии редактора карты автоматически включаются режимы редактирования для WorldMap, WorldBlocking и WorldSurface.
  • Редактор ресурсов использует ResourcePalette.gd, отправляет RESOURCE_EDIT.
  • Локальные правки накапливаются и отправляются одной пачкой с глобальными ID тайлов.
  • После сохранения клиент запрашивает обновленные чанки только для затронутых координат.

Примечание: Редактор объектов (ObjectEditorPanel) удален из интерфейса.

Унифицированный редактор (UnifiedMapEditor)

Унифицированный редактор объединяет все пять режимов редактирования в одной панели с вкладками:

  • Terrain - редактирование тайлов карты (TilePalette).
  • Objects - размещение объектов (ObjectPalette).
  • Resources - размещение ресурсов (ResourcePalette).
  • Blocking - создание зон блокировки (BlockingPalette).
  • Surface - настройка типов поверхности (SurfacePalette).

Кнопки управления: Save (сохранить активный слой), Back (отменить изменения и выйти).

Блокировки (WorldBlocking.gd)

Система блокировок позволяет создавать зоны, непроходимые для игрока:

  • Блокировки загружаются с сервера по команде BLOCKING.
  • Каждая блокировка может занимать несколько тайлов (1x1, 2x2, 3x3 и т.д.).
  • В режиме редактирования блокировки отображаются как красная буква "B" (Color(1, 0.2, 0.2)) на полупрозрачном фоне.
  • Размер шрифта буквы: 20px, позиционируется по центру тайла.
  • Вне редактора блокировки невидимы, но проверяются при движении.
  • Проверка: is_world_blocked(world_pos) возвращает true если позиция заблокирована.

Типы блокировок описаны в data/blocking_types.json:

{
  "types": [
    { "typeId": "block_1x1", "displayName": "Block 1x1", "size": [1, 1], "color": "#FF000080" },
    { "typeId": "block_2x2", "displayName": "Block 2x2", "size": [2, 2], "color": "#FF000080" }
  ]
}

Поверхности (WorldSurface.gd)

Система поверхностей добавляет модификаторы движения для различных типов местности:

  • Поверхности загружаются с сервера по команде SURFACE.
  • Каждый тайл может иметь свой тип поверхности.
  • В режиме редактирования поверхности отображаются цветными буквами на полупрозрачном фоне:
    • "D" - Deep Water (глубина) - синий цвет
    • "S" - Shallow Water (мелководье) - голубой цвет
    • "W" - Water (вода) - средне-синий
    • "I" - Ice (лёд) - светло-голубой
    • "M" - Mud (болото) - коричневый
  • Размер шрифта буквы: 20px, позиционируется по центру тайла.
  • get_surface_at(world_pos) возвращает параметры поверхности:
// Возвращаемый словарь
{
  "surfaceId": 2,
  "speedMod": 0.3,    // множитель скорости
  "damage": 1.0,      // урон в секунду
  "blocking": true    // блокирует проход
}

Типы поверхностей описаны в data/surface_types.json:

{
  "types": [
    { "surfaceId": 0, "name": "ground", "displayName": "Ground", "speedMod": 1.0, "damage": 0, "blocking": false },
    { "surfaceId": 1, "name": "water_shallow", "displayName": "Shallow Water", "speedMod": 0.7, "damage": 0, "blocking": false },
    { "surfaceId": 2, "name": "water_deep", "displayName": "Deep Water", "speedMod": 0.3, "damage": 1, "blocking": true },
    { "surfaceId": 4, "name": "ice", "displayName": "Ice", "speedMod": 1.2, "damage": 0, "blocking": false },
    { "surfaceId": 5, "name": "mud", "displayName": "Mud", "speedMod": 0.4, "damage": 0, "blocking": false }
  ]
}

Ресурсы: единый PNG и два слоя

Ресурсы отображаются в WorldResources.gd как два TileMap-слоя: нижний (блокирующий) и верхний (визуальный, поверх игрока). Синхронизация идет через RESOURCES/RESOURCE_EDIT.

  • Каждый ресурс — отдельный PNG (любой размер, не обязательно кратен 32).
  • Базовая клетка ресурса = нижний левый тайл изображения.
  • Нижний ряд тайлов всегда непроходимый (без коллизий, только логическая блокировка).
  • Верхние ряды проходимые и рисуются поверх игрока (игрок всегда за ними, включая голову).
  • Блокировка считается по нижнему тайлу игрока (32x32 у ног).
  • После удаления ресурса проход полностью освобождается.

Структура data/resource_types.json:

{
  "types": [
    {
      "typeId": "tree_oak",
      "displayName": "Oak Tree",
      "maxAmount": 8,
      "gatherTool": "axe",
      "drops": [
        { "itemId": "wood_log", "min": 2, "max": 4 }
      ],
      "texture": "res://resources/tree_oak.png"
    }
  ]
}
  • texture - путь к PNG ресурса.
  • typeId должен совпадать с серверным resource_types.json.
  • gatherTool и drops используются при добыче.

Как добавить ресурс (шаги)

  • Положи PNG в res://resources/ (например, tree_oak.png).
  • Размер PNG может быть любым: клиент режет изображение на клетки 32x32.
  • Нижний ряд изображения считается основанием и всегда блокирует проход.
  • Верхние ряды проходимые и рисуются поверх игрока.
  • Добавь тип в data/resource_types.json и синхронно в server/resource_types.json.
  • В редакторе ресурсов новый тип появится автоматически в палитре.

Важно: базовая клетка ресурса = нижний левый тайл PNG. Если PNG шире 1 клетки, ресурс занимает несколько тайлов вправо.

Данные и ассеты

  • ui/boot_splash.svg - кастомная заставка на старте клиента.
  • data/object_types.json описывает список типов объектов для палитры.
  • resources/*.png - отдельные PNG для каждого ресурса.
  • Текстуры меню и загрузки: ui/main_menu_bg.png, ui/loading_screen.png.
  • Тайлы мира и объектов: tiles/terrain.png, tiles/objects.png.
