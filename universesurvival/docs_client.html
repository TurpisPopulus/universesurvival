Клиент: вход, меню, мир и редакторы

Клиент отвечает за UI, визуализацию мира и синхронизацию с UDP-сервером. Вся игра живет в сцене main.tscn, а взаимодействие с сервером происходит через короткие текстовые команды и JSON-снимки по версиям.

Godot 4.5

UI + Scene Главное меню и игровая сцена находятся в отдельных tscn.

UDP протокол

Регистрация, вход и синхронизация карты/объектов выполняются по UDP.

Редакторы

Внутри игры доступны админские панели редактирования карты и объектов.

Быстрый старт

  • Запусти сервер: server/run_server.bat.
  • Открой проект Godot и запусти сцену res://main_menu.tscn.
  • Используй Create Player для регистрации, затем Enter для входа.

Главное меню (main_menu.tscn + MainMenu.gd)

  • Enter открывает окно входа и отправляет LOGIN.
  • Create Player показывает форму регистрации и отправляет REGISTER.
  • Settings открывает экран настроек клиента.
  • Exit закрывает игру.

Окна входа/регистрации и настройки скрывают главное меню, чтобы не было наложений.

Настройки

Экран настроек содержит четыре кнопки: Видео, Звук, Авторы, Назад.

  • Видео: выбор доступного разрешения окна и переключение режимов Окно/Полный экран.
  • Звук: пока пусто (заготовка под будущие слайдеры громкости).
  • Авторы: краткая информация о проекте.
  • Назад: возврат в главное меню.

Список разрешений формируется из распространенных размеров, не превышающих размер экрана.

Игровая сцена (main.tscn)

  • WorldMap и WorldObjects отображают карту и объекты, WorldResources рендерит ресурсы в два слоя (низ/верх).
  • Player отвечает за движение и отправку позиционных пакетов.
  • Ui содержит админское меню и панели редакторов.
  • LoadingOverlay показывает загрузочный экран до полной синхронизации.
  • PauseOverlay открывается по ESC, ставит игру на паузу и содержит настройки.

Сеть и протокол

Клиент общается с сервером через UDP-сообщения:

PING
REGISTER|name|password|appearance
LOGIN|name|password
id|name|x|y|appearance
CHUNK|cx|cy|lastVersion
EDIT|{...}
OBJECTS|cx|cy|lastVersion
OBJECT_EDIT|{...}
RESOURCES|cx|cy|lastVersion
RESOURCE_EDIT|{...}
  • appearance - base64 JSON с внешностью, передается при регистрации/входе и в позиционных пакетах.
  • CHUNK и OBJECTS возвращают JSON-снимки только при изменении версии.

Внешность персонажа

Модуль CharacterAppearance.gd собирает paperdoll по наборам атласов:

  • Тело: body_skinny.png, body_normal.png, body_fat.png.
  • Головы, волосы, бороды, лицо: отдельные атласы с 6 кадрами анимации.
  • Настройки внешности сохраняются как base64 JSON в аккаунте.

Редакторы карты, объектов и ресурсов

  • Админская кнопка открывает меню редакторов внутри игры.
  • Редактор карты использует TilePalette.gd, отправляет EDIT при сохранении.
  • Редактор объектов использует ObjectPalette.gd, отправляет OBJECT_EDIT.
  • Редактор ресурсов использует ResourcePalette.gd, отправляет RESOURCE_EDIT.
  • Локальные правки накапливаются и отправляются одной пачкой.

Ресурсы: единый PNG и два слоя

Ресурсы отображаются в WorldResources.gd как два TileMap-слоя: нижний (блокирующий) и верхний (визуальный, поверх игрока). Синхронизация идет через RESOURCES/RESOURCE_EDIT.

  • Каждый ресурс — отдельный PNG (любой размер, не обязательно кратен 32).
  • Базовая клетка ресурса = нижний левый тайл изображения.
  • Нижний ряд тайлов всегда непроходимый (без коллизий, только логическая блокировка).
  • Верхние ряды проходимые и рисуются поверх игрока (игрок всегда за ними, включая голову).
  • Блокировка считается по нижнему тайлу игрока (32x32 у ног).
  • После удаления ресурса проход полностью освобождается.

Структура data/resource_types.json:

{
  "types": [
    {
      "typeId": "tree_oak",
      "displayName": "Oak Tree",
      "maxAmount": 8,
      "gatherTool": "axe",
      "drops": [
        { "itemId": "wood_log", "min": 2, "max": 4 }
      ],
      "texture": "res://resources/tree_oak.png"
    }
  ]
}
  • texture - путь к PNG ресурса.
  • typeId должен совпадать с серверным resource_types.json.
  • gatherTool и drops используются при добыче.

Как добавить ресурс (шаги)

  • Положи PNG в res://resources/ (например, tree_oak.png).
  • Размер PNG может быть любым: клиент режет изображение на клетки 32x32.
  • Нижний ряд изображения считается основанием и всегда блокирует проход.
  • Верхние ряды проходимые и рисуются поверх игрока.
  • Добавь тип в data/resource_types.json и синхронно в server/resource_types.json.
  • В редакторе ресурсов новый тип появится автоматически в палитре.

Важно: базовая клетка ресурса = нижний левый тайл PNG. Если PNG шире 1 клетки, ресурс занимает несколько тайлов вправо.

Данные и ассеты

  • ui/boot_splash.svg - кастомная заставка на старте клиента.
  • data/object_types.json описывает список типов объектов для палитры.
  • resources/*.png - отдельные PNG для каждого ресурса.
  • Текстуры меню и загрузки: ui/main_menu_bg.png, ui/loading_screen.png.
  • Тайлы мира и объектов: tiles/terrain.png, tiles/objects.png.
