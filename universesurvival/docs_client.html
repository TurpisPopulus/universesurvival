Universe Survival Client
Godot client · Stage 1

Клиент Godot, связанный с UDP сервером

Клиент отправляет позицию игрока на сервер, получает список игроков и данные карты. Есть базовый редактор карты для админа.

Что делает

Отправляет id, имя и координаты по UDP, принимает список игроков и чанки карты.

Что не делает

Нет интерполяции, предсказаний и прав доступа на изменение карты.

Формат пакета

id|name|x|y (UTF-8, разделитель - |).

Быстрый старт

Сначала запусти сервер, затем открой проект Godot.

cd server

dotnet run --project KiloServer.csproj

# Клиент
# Открой universesurvival/project.godot в Godot 4 и запусти сцену main_menu.tscn

Главное меню

Первой сценой запускается main_menu.tscn.

Кнопка Enter открывает окно входа с вводом имени и пароля.

Create Player открывает форму создания имени и пароля.

Регистрация и вход выполняются через сервер, локальное сохранение не используется.

Регистрация запрещает повторяющиеся имена.

Вход проверяет имя на сервере и не пускает, если оно уже онлайн.

Статус сервера отображается слева внизу (online/offline).

При входе игрок появляется в сохраненной на сервере позиции.

Сервер возвращает уровень доступа, он включает кнопку админки.

Settings пока неактивна.

Настройки подключения

Параметры настраиваются в инспекторе узла Player (экспортированные свойства).

@export var server_address: String = "127.0.0.1"
@export var server_port: int = 7777
@export var player_id: String = "player-1"
@export var player_name: String = "Ari"
@export var send_rate_hz: float = 10.0

При старте клиент присваивает случайный player_id.

Отправка позиции

Клиент отправляет мировую позицию игрока с точностью до 3 знаков.

player-1|Ari|12.500|8.750

Ответ сервера содержит несколько строк и сохраняется в _last_reply.

Карта и редактор

Карта запрашивает чанки у сервера сообщениями CHUNK|cx|cy|lastVersion.

Ответ - JSON сетка 100x100 с версией, которая рисуется в WorldMap.gd.

Админка открывает меню из 10 кнопок и доступ к редактору карты.

Редактор позволяет выбрать тайл и ставить его мышкой по карте.

Сохранение отправляет изменения на сервер через EDIT.

Клиент рисует карту только по данным, полученным от сервера.

Удаленные игроки

Сцена remote_player.tscn используется для отображения других игроков.

Создание и обновление выполняется в Player.gd по id.

Подсказка

Если нужно увидеть ответ на экране, можно добавить UI и выводить _last_reply в Label. Сейчас ответ доступен для отладки.
